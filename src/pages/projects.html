Programming Project: Password Generator/Analyzer

Passwords are a critical aspect of digital security, serving as the first line of defense for user authentication. In this programming project, the goal is to design a robust Password Generator/Analyzer program that not only generates strong passwords but also evaluates the strength of user-input passwords against predefined rules.

The recommended approach for constructing passwords involves using a combination of unrelated random yet memorable phrases, integrating alphanumeric characters and symbols. This strategy results in longer, more complex passwords that are both secure and easier to remember, reducing the risk of password compromise.

The program should prompt users to create their own password and include a validator to measure its strength. Rejected passwords should not meet the minimum requirements, ensuring robust security practices. Additionally, users can opt for the program to generate a very strong password automatically, exceeding minimum requirements.

Key Requirements:

  • Password Strength Feedback: The program should provide clear feedback on whether the user's password meets the minimum requirements.
  • Success Indication: When the password meets requirements, the program should display text and visual cues to indicate success.
  • Random Password Generation: The program should have the capability to generate a random password that exceeds minimum requirements.
  • User-Provided Input: Functionality should exist to generate a random password based on user-provided input, ensuring a strong and secure password.

Coding Assignment: Nutritional Chart Rating System

Purpose:

  • assignments serve as a practical application of theoretical knowledge, reinforcing concepts learned in lectures and readings. This assignment focuses on utilizing JavaScript principles to create a program that enhances web design skills. Students will implement a rating scheme for a provided Nutritional Chart, promoting interactive behavior using JavaScript.

Assignment Instructions:

  • Students are tasked with assigning ratings to the Nutritional Chart based on calorie and carbohydrate criteria. The program should allow users to submit queries to list foods with the highest or lowest ratings for calories or carbohydrates. Additionally, users should be able to filter the chart to display specific food groups based on ratings, such as only four (4) or five (5) starred foods.
  • Lists specific foods, their carbohydrate and calorie values, and corresponding ratings.
  • Rating rules are defined for both carbohydrates and calories.
  • Supported Queries:
    • List foods by Carbs - highest or lowest rating.
    • List foods by Calories - highest or lowest rating.

Key Requirements:

  • List foods grouped by a single rating (five (5) starred to one (1) starred).
  • Display ALL foods.

Display Requirements:

  • When listing food ratings for carbs, display: Food, Carbohydrates, Rating, Calories.
  • When listing food ratings for calories, display: Food, Calories, Rating, Carbohydrates.
  • Highlight the first three (3) items when a user filters the list by star or food.

This assignment challenges students to apply problem-solving skills and JavaScript API principles to create an interactive and visually appealing program that strengthens their understanding of web design concepts.
